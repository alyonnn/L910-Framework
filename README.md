#  L910-Framework — Cosmetic Store (Вариант 12)

##  Описание проекта
Данный проект представляет собой минималистичный веб-фреймворк для Node.js, разработанный «с нуля» без использования сторонних библиотек (Express, Koa и др.). 
Цель работы — изучить механизмы маршрутизации, работу с событиями (EventEmitter), потоками (Streams) и принципы создания middleware.

##  Технологии
* **HTTP Module**: Создание сервера и обработка запросов.
* **EventEmitter**: Реализация системы маршрутизации.
* **Streams**: Сборка и парсинг тела запроса (req.body).
* **FS (FileSystem)**: Синхронизация данных с JSON-файлами.

##  Сущности и типы данных (Вариант 12)

В соответствии с заданием, данные содержат типы: `string`, `number`, `boolean`, `Date string` и `Array`.

### 1. Косметические товары (Products) — `products.json`
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| **id** | `number` | Уникальный ключ товара |
| **name** | `string` | Название продукта |
| **price** | `number` | Цена товара |
| **isVegan** | `boolean` | Этичность (веганский продукт) |
| **arrivalDate** | `Date string` | Дата поступления на склад |
| **ingredients** | `Array` | Список активных ингредиентов |

### 2. Бренды (Brands) — `brands.json`
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| **id** | `number` | Уникальный ключ бренда |
| **title** | `string` | Название бренда |
| **rating** | `number` | Рейтинг популярности |
| **isPremium** | `boolean` | Относится ли к люкс-сегменту |
| **founded** | `Date string` | Дата основания компании |
| **categories** | `Array` | Категории товаров бренда |

##  API Эндпоинты

Все запросы протестированы через **Postman**.

###  Товары (`/products`)
* `GET /products` — Получить весь список товаров.
* `GET /products/:id` — Получить товар по его ID.
* `POST /products` — Добавить новый товар (автоматическая генерация ID и даты).
* `PUT /products/:id` — Полное обновление информации о товаре (идемпотентно).
* `PATCH /products/:id` — Частичное обновление данных (например, только цены).
* `DELETE /products/:id` — Удаление товара из базы.

###  Бренды (`/brands`)
* `GET /brands` — Список всех брендов.
* `GET /brands/:id` — Получить бренд по ID.
* **Аналогично реализованы**: POST, PUT, PATCH, DELETE.

##  Функционал фреймворка
1. **Middleware**: Поддержка цепочки функций-обработчиков (например, логирование).
2. **Body Parser**: Самописный парсер JSON-данных из потока.
3. **Status Codes**: Удобная установка статус-кодов через `res.status().json()`.
4. **Error Handling**: Обработка 404 ошибок и исключений при чтении файлов.

##  Инструкция по запуску
1. Выполните установку: `npm i`
2. Запустите сервер: `npm start`
